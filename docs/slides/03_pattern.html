Request Flow with Policy Enforcement
Application / AI Agent Makes a request (syscall / tool call)
Boundary Syscall Interface / Router
HOOK (Intercept) LSM Hook / Policy Engine
DENY Block request
Policy Evaluation Check against rules
ALLOW Proceed to resource
The Invariant (25 Years)
Insert hooks at the boundary → Evaluate against policy → Enforce decisions
Pattern Elements
Standard flow component
Security boundary (hook point)
25-Year Lineage
2001 - Linux Kernel (LSM)
Syscall hooks, SELinux security server
2008 - Virtualization (sVirt)
Libvirt security driver hooks
2014 - Containers (go-selinux)
Container runtime labels
2018 - Kubernetes (PSA)
Pod Security Admission webhooks
2026 - AI Agents (Golden Agent)
Tool invocation hooks in Router
Insight

Every time we move up the stack, we bring the pattern with us. The hook point changes, but the architecture remains constant: intercept at boundary, evaluate policy, enforce decision.
