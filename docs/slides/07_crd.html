Policy as Code - api/v1alpha1/agentpolicy_types.go (254 lines)

Example: Coding Assistant Policy
apiVersion: agents.sandbox.io/v1alpha1
kind: AgentPolicy
metadata:
  name: coding-assistant-policy
spec:
  agentTypes:
    - coding-assistant
    - code-reviewer
  defaultAction: deny            # Principle 3: Default Deny
  mode: enforcing                 # or "permissive" for rollout

  toolPermissions:
    - tool: file.read
      action: allow
      constraints:
        pathPatterns:              # Can only read /workspace
          - "/workspace/**"

    - tool: network.fetch
      action: allow
      constraints:
        allowedDomains:            # Only approved APIs
          - "api.github.com"
          - "pypi.org"

    - tool: shell.execute
      action: deny               # No shell access

  tenantIsolation:               # Multi-Tenant Sandboxing
    mtsLabel: "s0:c100,c200"
    enforceMode: strict

Insight

This CRD is the Kubernetes-native equivalent of SELinux .te policy files. Version controlled, auditable, declarative.

The kubebuilder markers generate OpenAPI validation - invalid policies are rejected at admission time, before they can cause problems.
